🧠 Prompt Engineering 核心技巧：思维链 (Chain-of-Thought)
思维链（CoT）提示通过在 prompt 中加入”让我们一步一步思考”（Let's think step by step）引导模型展示中间推理步骤，显著提升数学推理、逻辑推理等复杂任务的准确率。CoT 分为 Few-shot CoT（提供带推理步骤的示例）和 Zero-shot CoT（仅用触发句）。进阶技巧包括：自一致性（Self-Consistency）——生成多条推理路径后投票选最优结果；思维树（Tree of Thoughts）——在推理空间中搜索不同分支并回溯；以及 ReAct——将推理（Reasoning）与行动（Acting）交替进行，在知识密集型任务中效果更佳。
参考: Chain-of-Thought Prompting Elicits Reasoning in Large Language Models (Wei et al., 2022)
🤖 Tool Use / Function Calling 的进阶：自动化工具发现与编排
当 Agent 面对大量工具时（数十到数百个 API），传统的”把全部工具 Schema 塞入 prompt“变得不现实。进阶方案：① 工具检索（Tool Retrieval）——用小 embedding 模型将工具描述向量化，用户意图动态召回最相关的 top-k 工具（像 mini-RAG），大幅减少 prompt 中的工具定义量；② 工具组合推理（Tool Composition）——LLM 被赋予”使用工具的元能力”，将多个原子操作编排为工具链，如”生成图表” = 查询数据 API → 数据清洗 → 调用图表生成工具；③ 自动化 API 适配——基于 OpenAPI/Swagger 规范自动生成 LLM 可理解的工具描述，减少人工编写 Schema 的工作。Gorilla 和 NexusRaven 等专用模型在工具选择准确率上优于同等规模的通用模型。
参考: Gorilla: Large Language Model Connected with Massive APIs
