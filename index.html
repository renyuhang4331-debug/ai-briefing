🤖 Agent 的记忆系统设计
AI Agent 的记忆系统通常分为三层：① 工作记忆（Working Memory）——当前对话上下文和任务状态，相当于”当前思考的内容”，最直接但容量受上下文窗口限制；② 短期记忆（Short-term Memory）——近期交互历史的摘要或关键信息，通过滑动窗口或摘要压缩持久化到向量数据库；③ 长期记忆（Long-term Memory）——用户偏好、历史知识、经验教训等结构化存储，支持跨会话检索和更新。关键技术包括：记忆重要性评分（自动判断什么值得记住）、记忆衰减机制（按时间和使用频率加权）、记忆冲突消解（新旧信息矛盾时的处理策略）。好的记忆系统是 Agent 实现”个性化”和”持续学习”的关键。

💡 AI 产品的定价策略：Token 经济 vs. 价值定价
AI 产品定价面临独特挑战：成本（推理算力）和用户价值之间缺乏线性关系。目前三种主流策略：① 按用量计费（Token-based）——按实际消耗 token 收费，透明但用户难以预估成本，适合 API 产品；② 订阅制（SaaS-style）——固定月费不限量或用额度包，便于用户预算规划，但需要对用量分布建模确保盈利（需考虑 heavy user 的 GPU 成本）；③ 结果导向定价（Outcome-based）——按交付的有效结果收费（如 AI 客服按解决的工单量），与客户价值强对齐但计费复杂。建议策略：早期用订阅制降低用户决策门槛获取数据飞轮，中期引入用量额度分层定价，成熟期向结果导向演进。关键指标：毛利率（Gross Margin）——AI 推理成本占收入的百分比，建议控制在 30% 以内。
