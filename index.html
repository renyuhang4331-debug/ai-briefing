👁️ 多模态大模型的三种主流架构
当前多模态大模型主要有三种架构路线：① 视觉编码器+LLM 拼接式（如 LLaVA、GPT-4V）——用预训练的视觉编码器（ViT）将图像转为 token 序列，经投影层对齐后与文本 token 拼接，送入 LLM 生成回答；训练成本低，可利用现有 LLM 和视觉模型。② 原生多模态（如 Gemini、Chameleon）——从预训练阶段就用图文混合 token 训练，在架构层面统一处理文本和图像生成；上限高但训练成本巨大。③ 专家混合式（如 LLaMA-Vision）——多模态信号走不同的专家路径，在特定层融合；在计算效率和跨模态理解精度间取平衡。目前①路线性价比最高，是大多数开源方案的基座。

🧠 大模型的 Scaling Laws：规模定律的三个维度
Scaling Laws 是大模型时代的”第一性原理”。OpenAI 2020 年论文和 DeepMind 的 Chinchilla 论文共同揭示了模型性能与三个要素的关系：模型参数量（N）、训练数据量（D）和计算量（C）。关键结论：① 在给定计算预算下，存在最优的模型大小和数据量分配——Chinchilla 定律建议模型参数和训练 token 大致 1:20 的配比；② 很多大模型（如 GPT-3）实际上”训练不足”（undertrained），在更小的模型上投入更多数据效果更优；③ Scaling Laws 也推动了小模型技术的发展——通过更强的数据质量（数据过滤、合成数据）可以在较小模型尺寸上获得超出 Scaling Law 预测的性能。理解 Scaling Laws 有助于判断模型技术报告中的关键数据点。
参考: Training Compute-Optimal Large Language Models (Hoffmann et al., 2022)
